The piSchedule control commands follow the notation described on the pilight documentation:

Devices

The first step is to define the pilight devices.

 "devices": {
        "Television": {
            "protocol": [ "relay" ],
              "id": [{
              "gpio": 3
            }],
            "state": "off",
            "default": "off"
        }
  },

The first JSON key 'Television' is the id used by pilight to identify a specific device, the value of the name (in this case Television with a capital T) setting is the name communicated by pilight.

piSchedule uses those config.json details for scheduling the devices. For a complete list of the commands, the usage and grammar have a look at the piSchedule Features

piSchedule Job Examples

To show how a dedicated device can be controlled, here are some examples:

The example device is defined in config.json with:

    "Haustuer": {
            "protocol": [ "pollin" ],
            "id": [{
                    "systemcode": 21,
                    "unitcode": 1
            }],
            "state": "off"
    }

Simple and direct switching

Haustuer; on;
Haustuer; off;

Time delayed switching

Haustuer; on,+:01;  off,+:2:00

'on' is delayed by one second; after 2 minutes it will be switched off

Random Time delay

Haustuer; on,~01:00;

A random time between 'now' and 'in one hour' will be used to switch on

Using Sunrise/Sunset

Haustuer; on, sunrise, +:30:00;

Switch time is 30 minutes after sunrise

Haustuer; on, sunset, ~-:45:00; off,sunset, +3:15:00;

On/off switch is oriented at sunset, 'on' varies between 0 and 45 minutes before sunset, 'off' will be 3 hours and 15 minutes after sunset

Multiple switching with one command line

Haustuer; on, 19:45, ~:10:00; off, 21:05; on, 21:30; off, +2:00:00;

This has two time pairs to define time periods, first switched 'on' at 19:45 (with random time variation) and 'off' at 21:05; the second set is from 21:30 until 23:30

piSchedule command testing

With piSchedule activ, testing commands for the switches can be send from the browser web-GUI:

  http://{server}:{port}/control?{command}

Example 'command' z.B: Haustuer;on (no Anführungszeichen, only ascii characters!)

On the browser this example will show this response:

    >>control::Haustuer;on<<

Sending time values with the test command it will be converted and added to the 'Actual Day Schedule, that way it will be processed normally.

Note: Adding a command with this testing will not be added to the active .INI file and thus it will be lost with changing the 'Actual Day Schedule'.

Log Files

Day Log files

All executed commands are written to a log-file which is named with actual weekday, f.e. [Friday.log]

Info and System Logs

This files can be found in the piSchedule7/logs directory
